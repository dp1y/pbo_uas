üë• Aktor Sistem

üî¥ Admin
Pengguna dengan level tertinggi yang memiliki akses ke semua fitur sistem. Admin bertanggung jawab untuk mengelola data user dan memonitor laporan parkiran.
üîµ Kasir
Petugas parkir yang bertugas mencatat kendaraan masuk dan keluar, serta melihat kondisi parkiran real-time.

üìù Use Case Deskripsi

1. Login
Aktor: Admin, Kasir
Deskripsi: Pengguna memasukkan username dan password untuk masuk ke sistem.
Precondition: User terdaftar di database
Postcondition: User berhasil login dan masuk ke dashboard sesuai level
2. Manajemen User (Admin Only)
Aktor: Admin
Deskripsi: Admin dapat menambah, edit, hapus, dan melihat data pengguna sistem.
Sub Use Case:
  • Tambah User: Membuat user baru dengan username, nama, password, dan level
  • Edit User: Mengubah data user yang sudah ada
  • Hapus User: Menghapus user dari sistem
  • Lihat Data User: Melihat list semua user dan detailnya
3. Catat Kendaraan Masuk
Aktor: Kasir
Deskripsi: Kasir mencatat kendaraan yang masuk parkir dengan memasukkan nomor polisi, jenis kendaraan, dan tarif.
Precondition: Kasir sudah login
Postcondition: Data kendaraan masuk tersimpan dengan status "parkir"
4. Catat Kendaraan Keluar
Aktor: Kasir
Deskripsi: Kasir mencatat kendaraan yang keluar parkir dengan menginput nomor polisi.
Related Use Case: Perhitungan Tarif
Postcondition: Data kendaraan update dengan status "keluar" dan tarif dihitung otomatis
5. Lihat Kondisi Parkiran
Aktor: Admin, Kasir
Deskripsi: Menampilkan list semua kendaraan yang sedang parkir dan riwayat kendaraan yang sudah keluar.
Informasi: No. Polisi, Waktu Masuk, Petugas Masuk, Waktu Keluar, Petugas Keluar, Status
6. Cari Kendaraan
Aktor: Kasir, Admin
Deskripsi: Mencari kendaraan berdasarkan nomor polisi untuk melihat detail dan status parkiran.
Precondition: User memasukkan nomor polisi
7. Perhitungan Tarif
Aktor: System (Otomatis)
Deskripsi: Sistem secara otomatis menghitung tarif parkir berdasarkan durasi dan jenis kendaraan.
Formula:
  • Pribadi: Durasi (jam) √ó Tarif per jam (Motor: Rp2.500, Mobil: Rp5.000)
  • Bus: Rp10.000 + (kelebihan penumpang / 5) √ó Rp2.500
8. Laporan Parkiran
Aktor: Admin
Deskripsi: Admin dapat melihat laporan kondisi parkiran yang mencakup semua transaksi hari ini.
Related Use Case: Lihat Kondisi Parkiran

üîÑ Skenario Alur Proses

Alur: Kendaraan Masuk

  1. Kasir membuka form "Kendaraan Masuk"
  2. Input nomor polisi kendaraan
  3. Pilih kategori: Pribadi atau Bus
  4. Jika Pribadi: pilih tipe (Motor/Mobil), tarif otomatis muncul
  5. Jika Bus: input kapasitas, tarif otomatis dihitung
  6. Tekan tombol "Simpan"
  7. Sistem catat waktu masuk dan nama petugas
  8. Status kendaraan: "parkir"

Alur: Kendaraan Keluar

  1. Kasir membuka form "Kendaraan Keluar"
  2. Input nomor polisi kendaraan
  3. Tekan tombol "Cari"
  4. Sistem tampilkan detail: no pol, waktu masuk, durasi, tarif
  5. Tekan tombol "Keluar"
  6. Sistem catat waktu keluar dan nama petugas
  7. Status berubah menjadi "keluar"
  8. Transaksi selesai

Alur: Manajemen User (Admin)

  1. Admin membuka menu "Kelola User"
  2. Tampil list semua user
  3. Pilih aksi: Tambah, Edit, atau Hapus
  4. Jika Tambah: input username, nama, password, level
  5. Jika Edit: pilih user, ubah data yang diinginkan
  6. Jika Hapus: konfirmasi, user dihapus dari sistem
  7. Tekan "Simpan"
  8. Sistem update database

Alur: Lihat Kondisi Parkiran

  1. Admin/Kasir membuka menu "Kondisi Parkiran"
  2. Sistem tampil list kendaraan hari ini
  3. Dapat melihat: no pol, waktu masuk, status, dsb
  4. Opsional: input keyword untuk cari nopol
  5. Tekan tombol "Cari"
  6. List ter-filter sesuai keyword
  7. Dapat melihat detail setiap kendaraan
